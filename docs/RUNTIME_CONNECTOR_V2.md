# Runtime Connector V2

Runtime Connector V2 lets an external agent framework register with Hermes Mission Control without exposing filesystem access or command execution by default.

For Hermes v0.16+ runtimes, connector records should prefer an adapter-safe contract over direct dashboard route coupling. A runtime may report `native_console_url`, `remote_gateway_url`, `dashboard_auth_mode`, `hermes_version`, profile identity, and capability flags for managed files, cron, Kanban, async delegation, approvals, and remote artifacts. Mission Control should normalize those fields through a `HermesRuntimeAdapter` boundary before exposing them to UI/domain flows.

## Public connector endpoints

All connector endpoints require a bearer token generated by a Mission Control operator:

- `POST /api/runtime-connect/register`
- `POST /api/runtime-connect/heartbeat`
- `POST /api/runtime-connect/events`

Admin/operator endpoints require Mission Control auth:

- `GET /api/runtime-connect`
- `POST /api/runtime-connect/tokens`
- `POST /api/runtime-connect/tokens/<token_id>/revoke`

## Connect a runtime

Create a token in Mission Control → Runtimes → Runtime Connector V2. Copy the token once.

Register:

```bash
curl -s -H "Authorization: Bearer $HMC_RUNTIME_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{
    "framework":"openclaw",
    "runtime_id":"friend-openclaw",
    "name":"Friend OpenClaw",
    "owner":"friend",
    "version":"0.1",
    "capabilities":{"controllable":false},
    "evidence":{"host":"friend-laptop","note":"local connector"}
  }' \
  https://hermes.melverick.com/api/runtime-connect/register
```

The response returns the concrete `runtime_id` to use for heartbeat/events.

Heartbeat:

```bash
curl -s -H "Authorization: Bearer $HMC_RUNTIME_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{
    "runtime_id":"openclaw-friend-openclaw-rtok-xxxxx",
    "status":"online",
    "capabilities":{"controllable":false},
    "evidence":{"jobs_running":0,"queue_depth":0}
  }' \
  https://hermes.melverick.com/api/runtime-connect/heartbeat
```

Event:

```bash
curl -s -H "Authorization: Bearer $HMC_RUNTIME_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{
    "runtime_id":"openclaw-friend-openclaw-rtok-xxxxx",
    "kind":"status",
    "severity":"info",
    "title":"OpenClaw connector online",
    "body":"No active jobs.",
    "payload":{"jobs_running":0}
  }' \
  https://hermes.melverick.com/api/runtime-connect/events
```

## Security defaults

- Tokens are stored as SHA-256 hashes; Mission Control cannot reveal a token again after creation.
- Remote runtimes are observability-first. They are not controllable unless the external connector explicitly advertises `capabilities.controllable=true`.
- Runtime-local artifacts must be represented with runtime-scoped locators such as `local_path`, `hermes_managed_file`, `runtime_file`, or `external_url`; do not assume a remote runtime path is readable from the HMC host.
- Policy/security blocks from Hermes should be preserved as policy states such as `blocked_by_policy` or `requires_policy_approval`, not collapsed into generic connector failure.
- Revoking a token blocks future register/heartbeat/event calls using that token.
- Heartbeats older than 180 seconds show as offline.
